# ModelForge:给小团队用的「小模型微调 + 大模型 Prompt 评测」一站式平台

> 开源地址:<https://github.com/LingyiChen-AI/ModelForge>

这半年我一直在折腾同一个问题:**不是所有 NLP 任务都值得上大模型**。

情感识别、意图分类、文本分类、命名实体识别(NER)、句子相似度这类活儿,一个几十 MB 的 BERT 小模型微调一下,**又快、又便宜、还能本地离线跑**,效果反而更稳定可控。可真要把「小模型」这条路走顺,你会发现处处是坑:数据要管版本、训练要搭环境、评估要写脚本、上线要自己拼、线上坏例还得回流再训练……

于是我做了 **ModelForge**,把这一整条链路收进一个平台;后来又顺手把另一条常见链路也做了进来——**大模型 Prompt 评测**(用平台调外部 LLM API 给提示词做盲测,人工 + AI 双重打分;注意这条线**只评测、不微调大模型**)。

![ModelForge 架构总览](https://cdn.jsdelivr.net/gh/LingyiChen-AI/images/ModelForge/images/2026/06/modelforge-architecture.png)

## 它解决什么痛点

- **大模型不总是最优解,成本却很高**:很多场景小模型微调就够,没必要每条请求都甩给大模型 API——既贵又慢,还得联网。
- **通用 / 开源模型在你的领域不够准**:尤其是 embedding 向量模型,通用权重在垂直语料上召回总差一截,需要拿自己的数据微调才好用,而微调本身门槛就高。
- **自建「训练 → 评估 → 上线」流程太碎**:数据、版本、训练、评估、部署、坏例回流,每一步都要自己拼脚本、搭环境。
- **Prompt 效果对比根本不好做**:换个 Prompt、换个模型,到底变好还是变坏?多半靠肉眼比对 + 手写脚本跑批,没有盲测、没有统一指标、结果难复现也难沉淀。

## 核心功能

### 1. 数据集 · 版本管理

训练集 / 评估集统一管理,每次提交都生成**不可变的全量快照**(parquet 落对象存储、带 checksum),保证训练随时可复现。支持 **CSV / JSONL / Excel** 导入,并能按任务类型一键下载对应格式的数据模板,省去对齐字段的来回试错。

### 2. 模型训练:四类任务,开箱即用

按任务类型分流训练 recipe,**文本分类、NER、句对相似度、检索向量(embedding)四类均已落地**。提交时先选基础模型(BERT / BGE / GTE 等分组目录),数据集**级联联动**——只列出格式匹配的版本,选错不了。训练过程有**实时进度条**,还能深链到 MLflow 看逐步 loss 曲线,实验与产物全程留痕。

### 3. 模型测试 · Leaderboard · 预测结果导出

对「模型版本 × 测试集版本」批量推理并自动算指标;在**同一测试集上横向对比**多个模型版本,就是一张现成的 Leaderboard。每次测试的**逐条预测都能一键导出成 Excel 表格**,真实标签与预测结果逐行对照,四类任务各按特点出列,复盘和交付都方便。

![模型测试:Leaderboard + 逐条预测导出](https://cdn.jsdelivr.net/gh/LingyiChen-AI/images/ModelForge/images/2026/06/modelforge-model-test.png)

![导出的预测结果表格](https://cdn.jsdelivr.net/gh/LingyiChen-AI/images/ModelForge/images/2026/06/modelforge-export.png)

### 4. 模型版本管理 · 一键在线部署

复用 MLflow Model Registry 做版本管理,支持手动**提升生命周期阶段**(未发布 / 预发布 / 生产 / 已归档)。满意的版本一键部署到推理服务,按任务类型自动暴露 `/predict`(分类、NER)、`/embed`(向量)、`/similarity`(句对)接口;每个部署都附带 **API 详情**(cURL 示例 + 输入输出说明),复制即用,还能随时停止 / 重启。

### 5. Badcase 闭环修复

线上业务通过接口(`X-Api-Key` 鉴权)把坏例回传,平台按模型版本归集。在**数据标注工作台**里逐条标注(主从两栏、未标注 / 未修复 / 已修复分桶、标注人与时间留痕),标完一键生成 `badcase-` 训练集回流再训练。再训练时还会**自动给这些坏例打分**,把已修复的标回去,并在指标里给出 **badcase 修复率**——形成「上报 → 标注 → 再训练 → 验证修复」的闭环。

### 6. 大模型 Prompt 评测(差异化亮点)

接入 LLM 供应商(OpenAI 协议)→ 把提示词做成**可版本管理的不可变模板**(`{{ 参数 }}` 占位)→ 复用数据集当测试集发起评测(多模型盲测 / 单 Prompt 版本对比)。

最关键的是**盲测设计**:界面只显示 A / B / C、隐藏版本,而且 A/B/C 在整轮内**固定对应同一候选**(避免你被版本号带偏),版本只在统计页揭晓——「我每条都选 A」就等于「始终偏好同一个候选」,结论可解释。除了人工盲评,还能让某个 LLM 当「评判员」做 **AI 自动评估**(强制 JSON 输出、并发执行、进度实时可见),与人工结论并存、互不覆盖;统计页同时给出**人工 + AI 两套指标**(各候选胜率 / 最优、单 Prompt 好评率与对上一版本的变好 / 变坏率)。

![Prompt 评测列表:人工 + AI 双评、进度可见、一键导出](https://cdn.jsdelivr.net/gh/LingyiChen-AI/images/ModelForge/images/2026/06/modelforge-prompt-eval-list.png)

![盲测工作台:只见 A/B/C,版本隐藏](https://cdn.jsdelivr.net/gh/LingyiChen-AI/images/ModelForge/images/2026/06/modelforge-prompt-blind-eval.png)

### 7. 认证与 RBAC

JWT 登录 + 自定义角色(在固定权限目录上自由组合)+ 角色级**数据范围**(`all` / `own`,`own` 只看 / 改自己创建的资源);超管管理用户与角色。机器对机器调用走 `X-Api-Key`,内部回调走 `X-Internal-Token`,边界清晰。

## 技术栈

- **后端**:FastAPI(app-server)+ Celery(ml-worker,跑训练 / 评估 / Prompt 评测 / Badcase 打分)+ FastAPI(model-server,在线推理)
- **前端**:React + TypeScript + Vite + Tailwind
- **基础设施**:PostgreSQL / Redis / MinIO / MLflow,四服务职责单一、解耦易扩展

## 快速开始

```bash
git clone https://github.com/LingyiChen-AI/ModelForge.git
cd ModelForge
docker compose up -d            # PG / Redis / MinIO / MLflow
# 然后按 README 安装依赖、初始化数据库、启动三端服务
```

完整步骤见仓库 README:<https://github.com/LingyiChen-AI/ModelForge>(Apache-2.0 开源)。

## 写在最后

ModelForge 适合**有自己数据、想把小模型这条路走顺**的小团队:不追求大而全,只把「数据 → 训练 → 评估 → 部署 → 闭环」和「Prompt 盲测评估」这两件最常用的事做扎实。欢迎到仓库点 Star / 提 Issue,也欢迎拍砖:<https://github.com/LingyiChen-AI/ModelForge>
