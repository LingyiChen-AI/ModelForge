
# 大模型prompt评测功能开发

## 设置页

添加设置页， 里面可以配置大模型的 base-url, api-key, model-id 字段， 还可以开启/关闭， 需要显示创建者， 配置完成后要加一个模式测试按钮， 点击可以测试模型回答是否正确， 所有模型按照 openai的模型协议接入。设置本身也要做入权限控制里面。

## Prompt页

添加prompt页面， 里面用户可以添加 prompt，prompt 是system prompt， 和 user prompt， 参数的格式一定是 {{ 参数 xx }}， 用户填写的时候，需要校验参数的写法， prompt 需要有版本管理 

测试模块需要新加三个类型：
1. 多 prompt 评测
可以选择多个 prompt， 选择多个 测试数据集。一个模型， 提交后， 对每个 prompt 进行测试， 任务需要展示进度， 跑完后， 需要展示每个 prompt 的评估结果， 评估的结果需要展示列表， 可以手动选择好/坏， 可以筛选好坏。 评估的时候是盲测，多个 prompt 对比，比如 A，B，C 三个 prompt 跑任务， 那么评估页里面就是三个 prompt 输出的答案，按钮就是 A更好， B更好， C更好，或者ABC一样坏，评估完成后，做出每个 prompt 的统计和最好 prompt 是哪个。数据集里面需要新增一个 prompt 测试集。 评估的时候选择的就是 prompt 测试集， 提交任务的时候需要校验prompt 测试集的字段和参数能否一一对应， 如果没有需要提示测试集缺少 xx 参数。

2. llm 模型评测
评估的时候需要选择模型， 可以多选， 评估任务需要展示进度，一个 prompt， 多个模型， 跑完后， 需要展示大模型跑出来的结果， 我们需要对大模型的结果做评估， 评估的结果需要展示列表， 可以手动选择好/坏， 可以筛选好坏。 评估的时候是盲测，多个大模型对比，比如 A，B，C 三个模型跑任务， 那么评估页里面就是三个模型输出的答案，按钮就是 A更好， B更好， C更好，或者ABC一样坏。评估完成后，做出每个大模型的统计和最好大模型是哪个，数据集里面需要新增一个 prompt 测试集。 评估的时候选择的就是 prompt 测试集， 提交任务的时候需要校验prompt 测试集的字段和参数能否一一对应， 如果没有需要提示测试集缺少 xx 参数。

3. 单 prompt 评测
选择某一个 prompt 版本 和多个测试集和一个模型， 使用模型跑出结果后， 自动去查上一次同一个 prompt 的上一个版本的数据， 做出变好率，变坏率是多少的统计，说明更好的是哪个版本。

大模型跑完的数据状态都是 待评估， 人工评估后就是已评估状态， 需要记录评估人和评估时间

# 大模型AI评估
设置一个按钮， 点击可以选择一个大模型， 将当前所有待评估的数据进行 AI 评估， 选出AI 认为最好的模型输出， 选择的模型的 API Key 需要保存在设置页面， AI 评估的 prompt 做到后台设置里面， 评估的数据需要显示是 AI 评估还是人工评估，AI 评估的同时需要显示模型输出的结果。
